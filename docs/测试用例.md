# 钢铁表面缺陷检测系统测试用例

## 1. 文档目的

本文档基于《钢铁表面缺陷检测系统-需求规格说明书》中的测试阶段内容，整理形成可执行、可追溯的测试用例库，用于验证系统在单元、集成、性能和用户验收层面的功能完整性与质量指标达标情况。

## 2. 测试范围

覆盖以下对象：

- `src/base_detector.py`
- `src/detection_engine.py`
- `src/vlm_engine.py`
- `src/camera.py`
- `src/db_manager.py`
- `src/exporter.py`
- `app.py`
- `cli.py`
- `skills/yolo-detect/`
- `skills/vlm-detect/`
- `scripts/benchmark.py`
- `scripts/prepare_dataset.py`
- `scripts/train_yolo.py`
- `scripts/log_analyzer.py`

## 3. 需求追溯

| 需求编号 | 需求说明 | 对应测试类别 |
| --- | --- | --- |
| FR-01 | 实时检测 | 集成、性能、UAT |
| FR-02 | 双引擎检测 | 集成、UAT |
| FR-03 | 人工审核 | 集成、UAT |
| FR-04 | 统计报表 | 集成、UAT |
| FR-05 | 数据导出 | 集成、UAT |
| FR-06 | 自然语言交互 | 集成、UAT |
| FR-07 | 图像采集 | 单元、集成、UAT |
| NFR-01 | 单张图像推理延迟 < 200ms | 性能 |
| NFR-02 | 检测 mAP >= 85% | 性能/算法验证 |
| NFR-03 | 系统可用率 >= 99.5% | 稳定性/长稳测试 |
| NFR-04 | API Key 不硬编码，沙箱隔离 | 安全性 |
| NFR-05 | 模块化架构支持扩展 | 集成/回归 |
| NFR-06 | 视频流处理帧率 >= 15 FPS | 性能 |

## 4. 测试环境

### 4.1 软件环境

- Windows 10/11
- Python 3.8+
- CUDA 11.x
- PyTorch、Ultralytics、OpenCV、Gradio、PyYAML、pytest

### 4.2 硬件环境

- 工控机或等效测试主机
- NVIDIA RTX 4060 8G 或等效 GPU
- 工业相机或 RTSP 流模拟源
- 可选 PLC 触发模拟器

### 4.3 测试数据

- 真实产线图像不少于 200 张
- 正常无缺陷图像不少于 50 张
- 覆盖裂纹、划痕、氧化皮、压痕、气泡、斑块等典型缺陷
- 覆盖边界场景：小目标、重叠缺陷、低照度、反光、运动模糊

## 5. 用例编写规则

- `P0`：核心业务必须通过，否则不可发布
- `P1`：主要功能，影响关键业务流程
- `P2`：增强功能，可在有限条件下延期修复
- 所有用例均需记录实际结果、是否通过、缺陷编号和执行人
