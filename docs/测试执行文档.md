# 钢铁表面缺陷检测系统测试执行文档

<!-- markdownlint-disable MD060 -->

## 1. 文档目的

本文档用于指导测试人员按统一流程执行测试、记录结果、提交缺陷和输出测试报告，确保测试活动可追踪、可复核、可签收。

## 2. 测试目标

- 验证系统是否满足需求规格说明书中的功能与非功能要求
- 验证核心链路是否稳定可用
- 验证性能指标是否达到上线门槛
- 验证异常场景下系统是否具备可恢复能力
- 为上线决策提供依据

## 3. 测试阶段划分

1. 单元测试
2. 集成测试
3. 系统测试
4. 性能测试
5. 用户验收测试（UAT）
6. 回归测试

## 4. 角色与职责

| 角色 | 职责 |
|---|---|
| 测试负责人 | 制定计划、分派任务、汇总结果、组织评审 |
| 测试执行人 | 按用例执行测试、记录证据、提交缺陷 |
| 开发人员 | 修复缺陷、配合定位问题、提供回归说明 |
| 产品/业务代表 | 参与 UAT、确认业务流程是否满足要求 |
| 运维人员 | 配置环境、协助部署、验证自启和监控 |

## 5. 测试前置条件

- 代码已完成目标版本冻结
- 模型权重文件已准备好
- 数据库文件可初始化
- 测试图片和标注数据已就绪
- API Key、相机、RTSP、PLC 等外部依赖已准备测试账号或模拟环境
- 已确认测试环境与生产环境的差异

## 7. 执行顺序

### 7.1 单元测试

执行范围：

- `tests/unit/test_detector.py`
- `tests/unit/test_db_manager.py`
- 其余单元测试按模块补充

推荐命令：

```bash
pytest tests/unit -v --cov=src --cov-report=term-missing
```

通过标准：

- 核心模块覆盖率不低于 80%
- 所有单元测试通过
- 无阻断级缺陷

### 7.2 集成测试

执行范围：

- 采集、检测、存储链路
- 检测、审核、报表链路
- 导出链路
- Skill 调用链路

推荐命令：

```bash
pytest tests/integration -v
python cli.py --mode detect --image data/images/sample_steel.jpg
python cli.py --mode export
```

通过标准：

- 核心流程端到端可跑通
- 数据落库与导出结果正确
- 异常场景有明确提示

### 7.3 性能测试

推荐命令：

<!-- markdownlint-disable MD013 -->
```bash
python scripts/benchmark.py --model models/weights/yolov8n.pt --iterations 100 --img-size 640 --batch-sizes 1,4,8 --device cuda:0
```
<!-- markdownlint-enable MD013 -->

监控项：

- 推理延迟 P50 / P95 / P99
- 吞吐量 FPS
- GPU 利用率
- 显存占用
- 内存占用

通过标准：

- P50 < 150ms
- P99 < 300ms
- 稳定 15 FPS 以上
- 无明显内存泄漏

### 7.4 UAT

参与人：

- 质检员
- 主管
- 运维代表

执行方式：

- 以真实业务流程操作
- 以截图、导出文件、日志为证据
- 逐条确认是否满足验收标准

## 8. 详细执行步骤

### 8.1 单元测试执行

1. 安装依赖
2. 准备临时数据库和测试图片
3. 运行单元测试
4. 记录失败用例
5. 对失败用例进行定位
6. 修复后回归

### 8.2 集成测试执行

1. 启动数据库和检测服务
2. 打开 Web 工作台或 CLI
3. 输入测试图像
4. 检查检测结果是否正确展示
5. 执行审核操作
6. 检查数据库中的审核字段
7. 导出 CSV、HTML 和 Bad Case
8. 检查导出文件内容

### 8.3 性能测试执行

1. 固定测试图像与模型版本
2. 预热模型
3. 运行基准测试脚本
4. 记录每轮耗时与资源占用
5. 对异常波动进行复测
6. 汇总性能结果

### 8.4 UAT 执行

1. 准备现场测试账号
2. 按业务场景逐条执行
3. 记录业务代表反馈
4. 对阻断问题立即关闭或标记风险
5. 形成验收结论

## 9. 测试数据管理

### 9.1 数据分组

| 数据组 | 用途 | 说明 |
|---|---|---|
| 正常图像 | 误报验证 | 不含缺陷 |
| 单缺陷图像 | 基础检测验证 | 每类缺陷至少有样本 |
| 多缺陷图像 | 复杂场景验证 | 多目标、多类别 |
| 边界样本 | 误检漏检分析 | 小目标、模糊、遮挡、反光 |
| 回归样本 | 缺陷修复验证 | 过去出现问题的样本 |

### 9.2 数据使用规则

- 测试数据与训练数据要区分标识
- 导出数据不得覆盖原始样本
- 含个人或敏感信息的数据不得外传
- 每次测试保留数据版本号
